### （一）网络安全法律法规

在渗透测试过程中，相关人员必须严格遵守国家关于网络安全的法律法规。以下列举了五部具有代表性的法律法规，并针对部分法律条款进行了重点说明，供参考与学习。

------

##### **1. 《中华人民共和国网络安全法》**

**立法目的**：

保障网络安全，维护网络空间主权和国家安全、社会公共利益，保护公民、法人及其他组织的合法权益，促进经济社会信息化健康发展。

**重点条款举例**：

- 第二十七条
  - 任何个人和组织不得从事非法侵入他人网络、干扰他人网络正常功能、窃取网络数据等危害网络安全的活动。
  - 不得提供专门用于侵入网络、干扰网络正常功能及防护措施、窃取网络数据等危害网络安全活动的程序、工具。
  - 明知他人从事危害网络安全的活动的，不得为其提供技术支持、广告推广、支付结算等帮助。

------

##### **2. 《中华人民共和国刑法》**

**重点条款举例**：

- **第二百八十五条**
  - 违反国家规定，侵入国家事务、国防建设、尖端科学技术领域的计算机信息系统的，处三年以下有期徒刑或者拘役。
  - 侵入其他计算机信息系统、获取数据、非法控制系统，情节严重的，处三年以下有期徒刑或者拘役，并处或单处罚金；情节特别严重的，处三年以上七年以下有期徒刑，并处罚金。
  - 提供专门用于侵入、非法控制计算机信息系统的程序工具，或明知他人实施相关违法犯罪行为而提供程序、工具，情节严重的，依照上述规定处罚。
- **第二百八十六条**
  - 违反国家规定，删除、修改、增加、干扰计算机信息系统功能，造成系统不能正常运行，后果严重的，处五年以下有期徒刑或者拘役；后果特别严重的，处五年以上有期徒刑。

------

##### **3. 《中华人民共和国保守国家秘密法》**

**立法目的**：

为保守国家秘密，维护国家安全和利益，保障改革开放和社会主义建设事业的顺利进行而制定。

------

##### **4. 《中华人民共和国国家安全法》**

**立法目的**：

为了维护国家安全，保卫人民民主专政的政权和中国特色社会主义制度，保护人民根本利益，保障改革开放和社会主义现代化建设顺利进行，实现中华民族伟大复兴，根据宪法制定。

##### **5. 《中华人民共和国计算机信息系统安全保护条例》**

**立法目的**：

为保护计算机信息系统的安全，促进计算机的应用和发展，保障社会主义现代化建设的顺利进行而制定。

1. 带宽：（单位：b/s , kb/s , mb/s , Gb/s , Tb/s)

表示单位时间内某信道内所能通过的“最高数据率”

1. 吞吐量：单位时间内通过某个网络（后信道、接口）的数据量

带宽是最高速率，吞吐量表示实际数据率

1. 时延：数据在传输过程中的时间延时

1. 发送时延：主机或路由器发送数据帧所需要的时间

发送时延=数据帧长度（bit）/ 发送速率（bit/s） 

1. 传播时延：电磁波在信道中传播一定的距离，需要花费一定的时间（主要影响因素是传输的介质）

传播时延= 信道长度（M） /  电磁波在信道上传播的速率（m/s)

1. 处理时延：主机或路由器在收到分组信息时花费的时间。

1. RTT的变化，一定程度上防疫了网络拥塞程度的变化。
2. 有效数据率 = 数据长度 / （发送时间 + RTT)

##### 2  -  网络体系结构

1. OSI体系结构（是一个七层协议的体系结构）

| 层级       | 功能                         |
| ---------- | ---------------------------- |
| 应用层     | 为应用程序提供服务           |
| 表示层     | 数据格式化、数据加密         |
| 会话层     | 建立、管理和维护会话         |
| 传输层     | 建立、管理和维护端到端的连接 |
| 网络层     | IP选址及路由选择             |
| 数据链路层 | 提供介质访问和链路管理       |
| 物理层     | 物理硬件设施基础             |

市场化失败的原因：缺少实际的经验，最终以失败告终

1. TCP/IP一个四层的体系结构：应用层、运输层、网际层、网络接口层
2. 五层协议的体系结构：它结合了OSI和TCP/IP的优点。

三种体系机构对比：

| OSI        | TCP/IP     | 五层协议   |
| ---------- | ---------- | ---------- |
| 应用层     | 应用层     | 应用层     |
| 表示层     |            |            |
| 会话层     |            |            |
| 运输层     | 运输层     | 运输层     |
| 网络层     | 网际层、IP | 网络层     |
| 数据链路层 | 网络接口层 | 数据链路层 |
| 物理层     | 物理层     |            |

应用层：不同的网络应用需要有不同的应用层协议。

eg:域名系统DNS,支持万维网应用的HTTP协议，支持电子邮件的SMTP协议等，我们把应用层交互的数据单元称为报文（Message）。

运输层：主要负责两台主机中的进程之间的通信，提供通用的数据传输服务。

eg:传输控制协议（TCP）：提供面向连接、可靠的数据传输服务。

​     用户数据报文协议（UDP)：提供无连接的、尽最大努力的数据传输服务。

TCP与UDP协议都有固定的格式，数据在经过运输层时会更具所选择的运输协议在应用层传过来的数据的基础上加上对应协议的头部。

网络层：实现两个网络系统之间数据透明传输，具体包括路由选择、拥塞控制和国际互联等。

由于网络层使用了IP协议，因此分组也叫IP数据包，简称为：数据报。

数据在经过网络层的时候会加上IP协议的头部 。

数据链路层（链路层）：在相邻的两个节点之间传输数据时，将IP数据报组装成帧，在相邻的节点上传输帧，

每一帧包括数据和必要的控制信息。

物理层：利用传输介质为数据链路层提供物理连接，实现比特流的透明传输，传输单位为bit/s。

   物理层传输介质：电缆、双绞线、光纤.

数据链路层中：路由器只有三层：网络层、数据链路层、物理层

   		  帧头和帧尾的作用之一就是帧定界。

透明传输：数据链路层对上层交付的数据没有任何的限制

##### 6 - 帧的两种转发方式

**存储转发**：交换机在转发之前必须接收整个帧，并进行错误校验，确定无误后再向目的地址发送，转发的时延随帧长度不同而变化。

**直接交换**：采用基于硬件的交叉矩阵,对帧不做任何处理，直接发送。

以太网交换机时一种即插即用的设备，其内部的帧交换表是通过（自学习算法）自动的建立起来的。

**帧交换表的时效性**：当有机子被交换后帧交换表将从新学习并更新

##### 7 - 以太网生成树协议STP

![image-20260315192108861](C:\Users\Lenovo\AppData\Roaming\Typora\typora-user-images\image-20260315192108861.png)

添加冗余链路可以提高以太网的可靠性：再交换机B与C之间添加冗余链路。但冗余链路也会带来负面影响——形成网路环路。

以交换机使用**生成树协议STP**可以在增加冗余链路来提高网络可靠性的同时又避免网络环路带来的各种问题。

**广播风暴：**

广播帧会在闭环链路中反复转发，顺逆时针同时兜圈，消耗网络资源是的网络设法转发其他数据帧。

主机收到反复的广播帧消耗大量主机资源。

交换机的帧交换表震荡同一个MAC地址，记录在反复的震。

STP能使交换机自动计算并构建一个逻辑上设有环路的网络，其逻辑拓扑结构必须是树型的，最终形成的属性逻辑拓扑要确保连通整个网络。

C理：在数据帧传输时部分链路被STP阻塞，避免了链路避免形成闭环。

**IP 注：**

1、只有A、B、C类地址可疑分给网络中的主机和路由器的各种接口。

2、主机号为”全0“的地址是网络地址（eg：128.127.0.0）不能分给主机或者路由器的各种接口。

3、主机号为”全1“的地址是广播地址（eg：192.168.43.1），不能分配给主机或路由器的各种接口。

A类范围：1.0.0.1~126.255.255.254   网络最大号127，作为本地环回地址，所以最大可指派的地址为126.

B类范围：128.0.0.1~191.255.255.254 ， 可指派的最小的IP地址为：128.0.0.1，末尾为1可疑作为网关地址。

C类范围：192.0.0.1~223.255.255.254 , 在C类中同一个网咯中最多可加254台机子。

子网的划分：

典题：同一根网线直接相连的两台主机的IP地址分别为192.168.1.100 、192.168.2.100，请问使用使用什么方法可疑让这两台主机能够正常的进行数据通信？

答：把两台主机的子网掩码设置为：255.255.0.0

子网掩码：是一个32位地址，又叫网络掩码，地址掩码，他是用来指明一个IP地址的哪些位标识的是主机所在的子网，哪些位标识的是主机，子网掩码不能单独存在，它必须结合IP地址一起使用。子网掩码将某个IP地址划分成网络地址和主机地址两个部分。

网络地址= IP与子网掩码进行按位与计算

**传输控制协议TCP**

TCP是一种基于连接的可靠的稳定的无重复的传输协议。

![image-20260315193959274](C:\Users\Lenovo\AppData\Roaming\Typora\typora-user-images\image-20260315193959274.png)

**1、32位序列号（sec)**：每个包都包含序列号，序列号被系统初始化位某个随机值ISN后续在TCP报文段中序列号加上该报文段所携带数据的第一个字节在整个字节流中的偏移。

eg:1025~2048该数据报文序列号位ISN+1025.

**2、32位确认号（ack）**：目的主机放回确认号，使源主机知道某个或某几个报文段已被接收。

**3、四位首部长度：**由于TCP首部包含一个长度可变的选项部分，可以需要这么一个组来指定这个TCP报文段到底有多长。

**4、URG标志**：标识紧急指针是否有效。

**5、ACK标志：**表示确认号是否有效，我们携带ACK标识的TCP报文段为确认报文段。

**6、PSH标志**：提示接收端应用程序应从TCP接收缓存区中读走数据，为接收后续数据腾出空闲（若数据未读走，则会一直留存）

**7.RST标志**：标识要求对方重新建立连接，我们称携带RST标志的TCP报文段为复位报文段。

**8.SYN标志**：表示请求建立一个连接，我们称携带SYN标志的TCP报文为同步报文段。

**9.FIN标志**：表示通知对方，本端要关闭连接了，我们把携带FIN标志的TCP报文为结束报文。

**10.16位窗口大小**：是TCP控制流量的一个手段，这里说的窗口指的是：接收报文窗口。它告诉对方本端TCP接收缓存区还能容纳多少字节的数据，这样对方可以控制发送数据的速度。

**11.16位校验和**：有发送编填空，接收端对TCP报文执行CRC算法以校验TCP报文段在传输过程中是否损坏。（这个校验不仅包括TCP头部，也包括数据部分，这也是对TCP可靠传输的一个重要保障。）
